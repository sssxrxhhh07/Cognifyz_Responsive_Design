Syntax

Essential patterns

The four most-used media query patterns — copy and adapt these for your project.

M
Mobile-first (recommended) 
/* Base styles = mobile */
.card {
  display: block;
  width: 100%;
}

/* Scale up for larger screens */
@media (min-width: 768px) {
  .card { display: grid; }
}

@media (min-width: 1024px) {
  .card { grid-template-columns: 1fr 1fr 1fr; }
}
D
Desktop-first 
/* Base styles = desktop */
.nav-links {
  display: flex;
  gap: 2rem;
}

/* Override for smaller screens */
@media (max-width: 768px) {
  .nav-links { display: none; }
}

@media (max-width: 480px) {
  .hamburger { display: flex; }
}
R
Range queries 
/* Target a specific range only */
@media (
  min-width: 480px
) and (
  max-width: 767px
) {
  .sidebar { width: 240px; }
}

/* Modern syntax (CSS Level 4) */
@media (480px <= width <= 767px) {
  .sidebar { width: 240px; }
}
F
Fluid typography 
/* clamp() — no media query needed */
h1 {
  font-size: clamp(
    1.8rem,   /* min */
    5vw,      /* fluid */
    3.6rem    /* max */
  );
}

/* Fluid spacing too */
.section {
  padding: clamp(1.5rem, 5vw, 4rem);
}
Techniques

Responsive patterns

Six layout and styling techniques that work together with media queries.

📐

CSS Grid

Use repeat(auto-fit, minmax(200px, 1fr)) for a grid that adapts automatically without media queries.

🔄

Flexbox wrap

Flex containers with flex-wrap: wrap naturally reflow children as space shrinks.

✏️

Fluid type

clamp() scales font sizes smoothly between a minimum and maximum — no breakpoints needed.

🖼

Responsive images

Set max-width: 100% and use the srcset attribute to serve the right resolution per device.

☰

Navigation

Hide the full nav at small sizes with display: none and reveal a hamburger menu via a media query.

🎨

Container queries

The new @container rule lets components respond to their parent's size, not just the viewport.
